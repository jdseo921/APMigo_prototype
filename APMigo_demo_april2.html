Tracking and session controls

Approved apps only

Limit tracking to your approved editor, browser, or work tools list.

Visible session indicator

Keep the active-state badge visible at all times while tracking is on.

Break nudges

Show reminders when sustained input activity or bursts exceed your comfort threshold.

Pause on unsupported app

Auto-pause the session if focus leaves the approved app list.

Privacy and transparency

These controls make it clear what is monitored, where data stays, and what users can review or delete. For an input-monitoring product, these are core trust features rather than optional extras.

Show monitoring scope on startup

Explain exactly which inputs are counted and which content is never captured before any session begins.

Session data retention

Choose whether summaries are kept for 7 days, 30 days, or only until manual export.

Local-only storage

Keep raw session data on-device unless you explicitly export a summary.

One-click session review

Open a readable session summary that explains what was counted, for how long, and under which approved apps.

Delete raw session traces

Remove fine-grained timing traces after each session and retain only summary metrics.

Privacy change alerts

Highlight any policy or permissions change in the Updates screen before the next session starts.

What is counted

Keystroke and mouse activity rates within approved apps during active sessions only.

What is not stored

Typed content, message bodies, passwords, or screenshots by default.

User controls

Pause, stop, review, export, and delete are accessible from the primary interface.

Security stance

Default to least-privilege permissions and local processing whenever possible.

Transparency settings

Let users decide how actively APMigo can interrupt or steer a session. These controls make the difference between passive monitoring, advisory nudges, and stronger intervention behavior.

Allow break prompts during high strain

Show interruption-level prompts when pace, burst intensity, or imbalance suggest rising wrist stress.

Auto-pause after repeated risk spikes

Pause tracking when several high-risk windows happen in a row, then ask the user to consciously resume.

Require explicit consent before each tracked session

Display a short scope reminder before monitoring starts so users can confirm what will be counted.

Ask before exporting session summaries

Prevent silent sharing by requiring confirmation before any session data leaves the device.

Suppress interventions during focus mode

Keep monitoring visible but hold back popups and pauses during protected deep-work sessions unless risk becomes severe.

About APMigo

APMigo is a transparent activity-insights tool designed to help users understand typing and input pace without hiding when monitoring is active. Its design goal is to support healthier work habits by combining visible sessions, approved-app boundaries, and simple wrist-load context.

Visible sessions only Approved apps scope Review and deletion controls Wrist-health awareness

Mission

Help users build sustainable input habits with clear monitoring boundaries, readable session summaries, and actionable strain context instead of opaque surveillance.

Design principles

Transparency before tracking: session status and monitoring scope stay visible.
Privacy by default: typed content is not stored, and local review/deletion remains accessible.
Actionable wellbeing: pacing and balance signals should help users decide when to slow down, redistribute load, or pause.

Team and support posture

The APMigo team treats false positives, data-retention questions, and monitoring-scope concerns as first-class support issues. Release notes and settings should keep these details easy to inspect before any session begins.
